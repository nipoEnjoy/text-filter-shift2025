# Особенности реализации

## 1. Чтение фалов

Про размер входных файлов ничего не сказано в задаче, поэтому положим, что он может быть очень большим. 
Поэтому читать целиком сразу не стоит, чтение сделано блоками через буфер. ОС уже имеет оптимизации для I/O операций с файлами на диске.
В таком случае единственная оптимизация - это размер буфера (в зависимости от файловой системы), для сокращения количества системных вызовов. Это позволяет сделать реализация BufferedReader/BufferedWriter. 

## 2. Статистика

В задаче не до конца ясно, что статистика должна собираться разными способами в зависимости от опций -s -f или только выводиться кратко или полно.
Поэтому сделал полный сбор статистики и вывод в зависимости от опции (по стандарту краткая).

## 3. Типы данных

Про тип данных для вещественных чисел в задании ничего не сказано. Поэтому используется float в соответствии с именем выходного файла.  
Не думаю, что в данной задаче точность float или double будет критичной (не влияет на суть задачи).

Требования к расширяемости дополнительными типами данных не было, масштабные добавления типов не предполагаются. 
Поэтому все вокруг трех integer, float и string, без излишних абстракций, наследований.

# Сборка и запуск

## Требования

**Java 25+** (OpenJDK 25+ или совместимые)  
Проверить активную версию можно командой `java -version`.

## Сборка

Используется Gradle 9.2.0.
```shell
./gradlew build
```

При необходимости дать права на исполнение.
```shell
chamod +x ./gradlew
```

После сборки .jar файл будет находиться в `./build/libs`.

## Запуск

```shell
java -jar ./build/libs/TextFilter.jar [-p prefix] [-o output_path] [-a] [-s] [-f] input_file1 [input_file2 ...]
```
